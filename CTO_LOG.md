# CTO Debug Log

## 2026-03-27

### Bug: Transparent BG toggle — blank grey screen

**Симптом:** Нажатие на тоггл Transparent BG убивало всю сцену. Cmd+R не помогал — только ручной ввод URL.

**Корневые причины (3 штуки, все связаны):**
1. **React 19 StrictMode + R3F v9** — StrictMode делает double-mount. WebGL контекст, созданный при первом маунте, уничтожался, а при ремаунте Three.js получал контекст с дефолтными атрибутами. Убрал StrictMode.
2. **useState в тоггле вызывал React re-render** — каскадный ре-рендер R3F компонентов мог дестабилизировать WebGL. Заменил на `useRef` + прямое обновление DOM (zero re-renders).
3. **Canvas без fallback clear color** — Three.js/R3F создаёт WebGL контекст с `alpha: true` (hardcoded). Если `scene.background` ещё не загрузился (Suspense, HDRI loading), canvas становится прозрачным → видно HTML фон. Добавил CSS `canvas { background-color: #1a1a1e }` + `gl.setClearColor()` в `onCreated`.

**Ключевая подсказка пользователя:** "Cmd+R не помогает, только ввод URL вручную" — это указало на то, что проблема не в стейте, а в WebGL контексте, который переживает soft reload.

### Bug: Bottom page edges too dark

**Симптом:** При экспорте нижние страницы книги почти чёрные, хотя в реальности страницы светлые.

**Причина:** Освещение сцены идёт сверху → нижняя грань (-y) получает минимум прямого света. Текстура с тёмными линиями (`rgba(0,0,0,0.58)`) усугубляла проблему.

**Решение:** `emissive={book.pageColor} emissiveIntensity={0.35}` на material-3. Emissive компенсирует нехватку света, не нарушая реалистичность.

### Bug: Export PNG отличается от браузера

**Симптом:** Скриншот из браузера vs сохранённый PNG — разные цвета, контраст, детализация. Экспорт темнее и менее контрастный.

**Причина:** `WebGLRenderTarget` по умолчанию имеет `texture.colorSpace = LinearSRGBColorSpace`. Рендерер НЕ применяет linear→sRGB конвертацию при рендере в такой target, в отличие от рендера на экран. Результат — пиксели в linear space, которые выглядят темнее и глуше.

**Решение:**
- `rt.texture.colorSpace = THREE.SRGBColorSpace` — рендерер теперь конвертирует в sRGB, как для экрана
- `samples: 4` — MSAA антиалиасинг, идентичный live canvas
